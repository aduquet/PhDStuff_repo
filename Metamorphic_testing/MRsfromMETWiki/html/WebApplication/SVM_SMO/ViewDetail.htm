Program Information

Name: SVM/SMO
Domain: Graph and Image
Functionality: A computer program to implement the SMO algorithm.
Input: A data point is a tuple ⟨xi→,yi⟩ in a K-dimensional space where xi→ is a vector that consists of K attributes and is described as [u(1)i,…,u(K)i]. Values of each attribute belong to a set D(u(j)i∈D). D may be a subset of real numbers R such as the interval [0,1]. yi is a label, and takes a value of -1 or +1, yi∈L where L={−1,+1}.A dataset X is a set of N data points such that X∈P(DK×L) and X={⟨xi→,yi}.
Output: We introduce translations T on datasets X such that T(X) returns a new dataset,T∈P(DK×L)→P(DM×L).We sometimes use a higher order function map as well, map∈(A→B)×P(A)→P(B) where A and B refer to certain domains. The function map(f,X) applies a function f of A→B to each element of the set X of P(A) to produce another set of P(B). We introduce translations Tc on datasets, essentially to increase the population of the input dataset. Tc adds data points so that a newly calculated hyperplane is predictable with respect to the current one. X(m) depicts the dataset obtained from the m iteration. Furthermore,we assume that the hyperplane is y(m)(x→) for the dataset X(m).We will present below how to obtain X(m+1) from X(m). X(m+1) contains more data points than X(m).

MR Information

MR1------Reorder Data Points

Description: MR for Pseudo Oracles
Property: A dataset X consists of N data points ⟨xn−→,yn⟩, X={⟨x1−→,y1⟩,…,⟨xi→,yi⟩,…,⟨xj→,yj⟩,…,⟨xN−→,yN⟩}. The translation T(1), for reordering data points, returns a dataset in which the data points with their indices i and j of the data points are swapped. T(1)(X)={⟨x1−→,y1⟩,…,⟨xj→,yj⟩,…,⟨xi→,yi⟩,…,⟨xN−→,yN⟩} where the indices i and j are arbitrary if they are between 1 and N and i≠j. For this translation T(1), the resultant Lagrange multipliers αi and αj are swapped as well.The hyperplane remains the same.

MR2------Reverse Labels 

Description: MR for Pseudo Oracles
Property: We introduce a function τ(2) on a data point, τ(2)∈DK×L→DK×L such that it returns a data point with a reversed label, τ(2)(⟨x→,l⟩)=⟨x→,−l⟩. Then, T(2)(X)=map(τ(2),X). For this translation T(2),he resultant Lagrange multipliers remain the same. Signs of the hyperplane are reversed, namely ω→ and b are now −ω→ and −b respectively.

MR3------Reorder Attributes 

Description: MR for Pseudo Oracles
Property: For a K-dimensional vector, x→=[u1,…,ui,…,uj,…,uK], we have a function τ(3) on a data point such that τ(3)(⟨[u1,…,ui,…,uj,…,uK],y⟩)=⟨[u1,…,uj,…,ui,…,uK],y⟩ where the indices i and j are arbitrary if they are between 1 and N and i≠j. Then, T(3)(X)=map(τ(3),X).  For this translation T(3), the resultant Lagrange multipliers remain the same, and so does the hyperplane.

MR4------Add a Constant Attribute 

Description: MR for Pseudo Oracles
Property: Let c be a constant ∈D. τ(4) essentially adds the constant c to form a K-dimensional vector x→ to be (K+1)-dimensional, τ(4)∈DK×L⇒DK+1×L. τ(4)(⟨[u1,…,uK],y⟩)=⟨[u1,…,uK,c],y⟩. Then, T(4)(X)=map(τ(4),X). For this translation T(4), the resultant  Lagrange multipliers may be changed, but the hyperplane remains the same. The newly added attribute does not have any effect on the classification of data points. Note that T(4) is of P(DK×L)→P(DK+1×L).

MR5------Change Attribute Values 

Description: MR for Pseudo Oracles
Property: Let op be an operation for changing attribute values, op∈D→D. τ(5) is a function to apply op to a particular attribute, τ(5)∈DK×L→DK×L. τ(5)(⟨[u1,…,ui,…,uK],y⟩)=⟨[u1,…,op(ui),…,uK],y⟩. Then,T(5)(X)=map(τ(5),X). This translation introduces invariants which depends on the choice of op and the kernel function. For example, if op is an addition of a constant to an attribute and the kernel function is Gaussian, then the resultant Lagrange multipliers and the hyperplane are unchanged.

MR6------Reduce Margin

Description: MR for Dataset Generation
Property: Tr is an instance of Tc with two additional data points ⟨x→N+1,+1⟩ and ⟨x→N+2,+1⟩ such that the both lie on the hyperplane to X, Tr∈P(DN×L)→P(DN+2×L). If the current hyperplane is such that y(m)(x→)=ω→(m)⋅x→+b(m), then y(m)(x→N+1)=0 and y(m)(x→N+2)=0. We denote X(m+1) to be Tr(X(m)), and that X(m+1)=X(m)∪{⟨x→N+1,+1⟩,⟨x→N+2,+1⟩}.
